## 프로젝트 소개
메타버스 아카데미 팀 구성 미팅에서 모델러는 개발자를 구하고 개발자는 기획자를 구하고 기획자는 모델러를 구하는 서로에 대한 니즈는 확실히 있지만 서로에 대한 이해가 부족해서 팀빌딩 과정에서 어려움을 느꼈습니다.

이러한 사람들이 팀빌딩을 단톡방을 통해서 하고 있는 것을 보고 기획자, 사업가, 개발자, 아티스트가 서로 공감대를 형성하지 않고 있다고 생각했습니다.

그래서 저희는 모든 직군들이 서로를 이해 할 수 있는 서비스를 만들어보자는 생각에 프로젝트를 기획하게 되었습니다.

## Commit message rule 
- feat : 새로운 기능 추가, 기존의 기능을 요구 사항에 맞추어 수정
- fix : 기능에 대한 버그 수정
- build : 빌드 관련 수정
- chore : 패키지 매니저 수정, 그 외 기타 수정 ex) .gitignore
- ci : CI 관련 설정 수정
- docs : 문서(주석) 수정
- style : 코드 스타일, 포맷팅에 대한 수정
- refactor : 기능의 변화가 아닌 코드 리팩터링 ex) 변수 이름 변경
- test : 테스트 코드 추가/수정
- release : 버전 릴리즈
